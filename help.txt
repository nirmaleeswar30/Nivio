Description:
    Generates an APK Set archive containing either all possible split APKs and
    standalone APKs or APKs optimized for the connected device (see connected-
    device flag).

Synopsis:
    bundletool build-apks 
        --bundle=<bundle.aab>
        --output=<output.apks>
        [--7zip]
        [--aapt2=<path/to/aapt2>]
        [--adb=<path/to/adb>]
        [--connected-device]
        [--country-set=<country_set_name>]
        [--create-stamp]
        [--device-id=<device-serial-name>]
        [--device-spec=<device-spec.json>]
        [--device-tier=<low>]
        [--fuse-only-device-matching-modules]
        [--key-pass=<key-password>]
        [--ks=<path/to/keystore>]
        [--ks-key-alias=<key-alias>]
        [--ks-pass=<[pass|file]:value>]
        [--lineage=<path/to/existing/lineage>]
        [--local-runtime-enabled-sdk-config=<path/to/config.json>]
        [--local-testing]
        [--max-threads=<num-threads>]
        [--min-v3-rotation-api-version=<30>]
        [--mode=<default|universal|system|persistent|instant|archive>]
        [--modules=<base,module1,module2>]
        [--oldest-signer=<path/to/keystore.properties>]
        [--optimize-
            for=<abi|screen_density|language|texture_compression_format|device_tier|country_set|ai_model_version|device_group>]
        [--output-format=<apk_set|directory>]
        [--overwrite]
        [--rotation-min-sdk-version=<30>]
        [--sdk-archives=<path/to/asar1.asar,path/to/asar2.asar>]
        [--sdk-bundles=<path/to/bundle1.asb,path/to/bundle2.asb>]
        [--stamp-exclude-timestamp=<true>]
        [--stamp-key-alias=<stamp-key-alias>]
        [--stamp-key-pass=<stamp-key-password>]
        [--stamp-ks=<path/to/keystore>]
        [--stamp-ks-pass=<[pass|file]:value>]
        [--stamp-source=<stamp-source>]
        [--store-package=<com.android.vending>]
        [--verbose]

Flags:
    --bundle: Path to the Android App Bundle to generate APKs from.

    --output: Path to where the APK Set archive should be created (default) or
        path to the directory where generated APKs should be stored when flag --
        output-format is set to 'DIRECTORY'.

    --7zip: (Optional) Path to the 7zip binary to use. Mandatory if Android App
        Bundle requires 7zip compression.

    --aapt2: (Optional) Path to the aapt2 binary to use.

    --adb: (Optional) Path to the adb utility. If absent, an attempt will be
        made to locate it if the ANDROID_HOME or PATH environment variable is
        set. Used only if connected-device flag is set.

    --connected-device: (Optional) If set, will generate APK Set optimized for
        the connected device. The generated APK Set will only be installable on
        that specific class of devices. This flag should only be set with --
        mode=default flag.

    --country-set: (Optional) Country set to use for APK matching. This flag
        should only be set with --device-spec or --connected-device flags. If a
        device spec with a country set is provided, the value specified here
        will override the value set in the device spec.

    --create-stamp: (Optional) If set, a stamp will be generated and embedded in
        the generated APKs.

    --device-id: (Optional) Device serial name. If absent, this uses the
        ANDROID_SERIAL environment variable. Either this flag or the environment
        variable is required when more than one device or emulator is connected.
        Used only if connected-device flag is set.

    --device-spec: (Optional) Path to the device spec file generated by the
        'get-device-spec' command. If present, it will generate an APK Set
        optimized for the specified device spec. This flag should be only be set
        with --mode=default flag.

    --device-tier: (Optional) Device tier to use for APK matching. This flag
        should only be set with --device-spec or --connected-device flags. If a
        device spec with a device tier is provided, the value specified here
        will override the value set in the device spec.

    --fuse-only-device-matching-modules: (Optional) When set, conditional
        modules with fused attribute will be fused into the base module if they
        match the device given by device-spec

    --key-pass: (Optional) Password of the key in the keystore to use to sign
        the generated APKs. If provided, must be prefixed with either 'pass:'
        (if the password is passed in clear text, e.g. 'pass:qwerty') or 'file:'
        (if the password is the first line of a file, e.g. 'file:
        /tmp/myPassword.txt'). If this flag is not set, the keystore password
        will be tried. If that fails, the password will be requested on the
        prompt.

    --ks: (Optional) Path to the keystore that should be used to sign the
        generated APKs. If not set, the default debug keystore will be used if
        it exists. If not found the APKs will not be signed. If set, the flag
        'ks-key-alias' must also be set.

    --ks-key-alias: (Optional) Alias of the key to use in the keystore to sign
        the generated APKs.

    --ks-pass: (Optional) Password of the keystore to use to sign the generated
        APKs. If provided, must be prefixed with either 'pass:' (if the password
        is passed in clear text, e.g. 'pass:qwerty') or 'file:' (if the password
        is the first line of a file, e.g. 'file:/tmp/myPassword.txt'). If this
        flag is not set, the password will be requested on the prompt.

    --lineage: (Optional) Input SigningCertificateLineage. This file contains a
        binary representation of a SigningCertificateLineage object, which
        contains the proof-of-rotation for different signing certificates. This
        can be used with APK Signature Scheme v3 to rotate the signing
        certificate for an APK. An APK previously signed with a
        SigningCertificateLineage can also be specified; the lineage will then
        be read from the signed data in the APK. If set, the flag 'oldest-
        signer' must also be set.

    --local-runtime-enabled-sdk-config: (Optional) Path to config file that
        contains options for overriding parts of runtime-enabled SDK dependency
        configuration of the app bundle. Can only be used if either 'sdk-
        bundles' or 'sdk-archives' is set.

    --local-testing: (Optional) If enabled, the APK set will be built in local
        testing mode, which includes additional metadata in the output. When
        `bundletool install-apks` is later used to install APKs from this set on
        a device, it will additionally push all dynamic module splits and asset
        packs to a location that can be accessed by the Play Core API.

    --max-threads: (Optional) Sets the maximum number of threads to use
        (default: 4).

    --min-v3-rotation-api-version: (Optional) The minimum API version for
        signing the generated APKs with rotation using V3 signature scheme.

    --mode: (Optional) Specifies which mode to run 'build-apks' command against.
        Acceptable values are
        'default|universal|system|persistent|instant|archive'. If not set or set
        to 'default' we generate split, standalone and instant APKs. If set to
        'universal' we generate universal APK. If set to 'system' we generate
        APKs for system image. If set to archive we generate the archived APK.

    --modules: (Optional) List of module names to include in the generated APK
        Set in modes universal and system.

    --oldest-signer: (Optional) Path to a properties file containing the
        properties of the oldest keystore that has previously been used for
        signing. This is used to sign the generated APKs with signature scheme
        v1 and v2 for backward compatibility when the key has been rotated using
        signature scheme v3. If set, the flag 'lineage' must also be set.
    
    The file is required to include values for the following properties:
      * ks - Path to the keystore.
      * ks-key-alias - Alias of the key to use in the keystore.
    
    The file may also optionally include values for the following properties:
      * ks-pass - Password of the keystore. If provided, must be prefixed with
        either 'pass:' (if the password is passed in clear text, e.g. 'pass:
        qwerty') or 'file:' (if the password is the first line of a file, e.g.
        'file:/tmp/myPassword.txt'). If this property is not set, the password
        will be requested on the prompt.
      * key-pass - Password of the key in the keystore. If provided, must be
        prefixed with either 'pass:' (if the password is passed in clear text,
        e.g. 'pass:qwerty') or 'file:' (if the password is the first line of a
        file, e.g. 'file:/tmp/myPassword.txt'). If this property is not set, the
        keystore password will be tried. If that fails, the password will be
        requested on the prompt.
    
    Example keystore.properties file:
    ks=/path/to/keystore.jks
    ks-key-alias=keyAlias
    ks-pass=pass:myPassword
    key-pass=file:/path/to/myPassword.txt

    --optimize-for: (Optional) If set, will generate APKs with optimizations for
        the given dimensions. Acceptable values are
        'abi|screen_density|language|texture_compression_format|device_tier|country_set|ai_model_version|device_group'.
        This flag should be only be set with --mode=default flag.

    --output-format: (Optional) Specifies output format for generated APKs. If
        set to 'APK_SET' outputs APKs into the created APK Set archive
        (default). If set to 'DIRECTORY' outputs APKs into the specified
        directory.

    --overwrite: (Optional) If set, any previous existing output will be
        overwritten.

    --rotation-min-sdk-version: (Optional) The minimum Android version for
        signing the generated APKs with rotation. The original signing key for
        the APK will be used for all previous platform versions.

    --sdk-archives: (Optional) Paths to SDK archives for the runtime-enabled
        SDKs that the App Bundle depends on, separated by commas. Each SDK
        archive must have an extension .asar. Can not be used together with the
        'sdk-bundles' flag.

    --sdk-bundles: (Optional) Paths to SDK bundles for the runtime-enabled SDKs
        that the App Bundle depends on, separated by commas. Each SDK bundle
        must have an extension .asb. Can not be used together with the 'sdk-
        archives' flag.

    --stamp-exclude-timestamp: (Optional) If set, a timestamp indicating the
        time of signing will be excluded from the generated stamp embedded in
        the generated APKs.

    --stamp-key-alias: (Optional) Alias of the stamp key to use in the keystore
        to sign the APK contents hash. If not set, the 'ks-key-alias' key alias
        will be tried if present.

    --stamp-key-pass: (Optional) Password of the stamp key in the keystore to
        use to sign the APK contents hash. if provided, must be prefixed with
        either 'pass:' (if the password is passed in clear text, e.g. 'pass:
        qwerty') or 'file:' (if the password is the first line of a file, e.g.
        'file:/tmp/myPassword.txt'). If this flag is not set, the keystore
        password will be tried. If that fails, the password will be requested on
        the prompt.

    --stamp-ks: (Optional) Path to the stamp keystore that should be used to
        sign the APK contents hash. If not set, the 'ks' keystore will be tried
        if present. Otherwise, the default debug keystore will be used if it
        exists. If a default debug keystore is not found, the stamp will fail to
        get generated. If set, the flag 'stamp-key-alias' must also be set.

    --stamp-ks-pass: (Optional) Password of the stamp keystore to use to sign
        the APK contents hash. If provided, must be prefixed with either 'pass:'
        (if the password is passed in clear text, e.g. 'pass:qwerty') or 'file:'
        (if the password is the first line of a file, e.g. 'file:
        /tmp/myPassword.txt'). If this flag is not set, the password will be
        requested on the prompt.

    --stamp-source: (Optional) Name of source generating the stamp. For stores,
        it is their package names. For locally generated stamp, it is 'local'.

    --store-package: (Optional) Package name of an app store that will be called
        by archived app to redownload the application. Play Store is called by
        default. Can only be provided for ARCHIVE mode.

    --verbose: (Optional) If set, prints extra information about the command
        execution in the standard output.
